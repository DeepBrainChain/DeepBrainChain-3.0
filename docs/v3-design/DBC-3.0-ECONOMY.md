# DBC 3.0 经济模型与奖励算法 (v1.0)

## 1. 节点质押 (Staking)
- **基础质押**: 每个 GPU 核心需质押固定数量的 DBC。
- **动态调节**: 当网络算力供过于求时，质押门槛自动上调；供不应求时，下调以吸引新节点。

## 2. 奖励分配公式
每个区块产生的 DBC 奖励 $R$ 分配给节点 $i$ 的比例为：
$$Reward_i = R \times \frac{Score_i \times V_i \times U_i}{\sum (Score_j \times V_j \times U_j)}$$

其中：
- $Score_i$: 硬件跑分值。
- $V_i$: ZK 验证成功率 (0.0 - 1.0)。
- $U_i$: 在线稳定性系数 (Uptime)。

## 3. 自动化惩罚 (Slashing)
- **轻微违规**: 离线超过 1 小时，扣除当日奖励。
- **严重违规**: ZK 验证发现计算作弊，没收 100% 质押金，并永久列入黑名单。

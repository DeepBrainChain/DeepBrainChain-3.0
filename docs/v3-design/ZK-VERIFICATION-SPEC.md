# DBC 3.0 ZK 验证技术规格书 (v1.0)

## 1. 验证目标
在不重新执行 AI 任务的情况下，通过数学证明确保节点 $i$ 确实运行了指定的模型 $M$ 并得到了结果 $R$。

## 2. 技术选型
- **证明系统**: Plonk (支持通用设置，适合多变的 AI 任务)。
- **算术电路**: 将常用的神经网络算子（如 Matrix Multiplication, ReLU）转化为 ZK 约束电路。
- **链上集成**: 在 Runtime 中集成 `pallet-zk-verify`，利用 `sp-io` 提供的加密原语。

## 3. 验证流程
1. **任务分发**: 链端下发任务哈希。
2. **本地计算**: 节点生成计算结果 $R$ 及 ZK 证明 $\pi$。
3. **提交验证**: 节点将 $(\pi, R)$ 提交至 `pallet-compute-pool`。
4. **链上校验**: Runtime 调用验证函数，成功则发放奖励，失败则触发 Slashing。
